Toad NUTS


An Android app (using C# and Xamarin) and web application (ASP.NET MVC) for an organization called Toad NUTS that focus on the conservation of the endangered Western Leopard Toad. The Android application enables the capturing of pictures and information (including GPS co-ordinates) of Western Leopard Toad sightings while the web application displays the captured data, creates statistics reports and plots sightings on a Google Map.

Additional features include the ability to setup areas by drawing polygons on Google Maps thereby enabling the system to automatically calculate and allocated toad sightings to specific areas. Areas are assigned to coordinators that receive automatic email notifications when toad sightings occur within their area. Pigeon Voice Middleware Server

Zebra ultimately discontinued the TekSpeech product range. I suggested and convinced one of our voice customers to take on an R&D endeavour by developing and providing them with our own middleware server to replace TekSpeech. Pigeon Voice Server is now our own white label middleware product that I personally designed and developed and have now implemented at all of our clients' voice sites around the country. It works with Honeywell Vocollect's proprietary message types (LUT & ODR) over TCP/IP (socket server) as well as over HTTP by providing a REST web service for the voice applications to consume. Pigeon Voice Website: developed a dashboard website in ASP.NET MVC to view statistics, devices and operators connecting as well as logs of every transaction going through the middleware. The server is a white label product that allows plugins to be developed for different WMS' that it connects depending on the business requirements. Whatsapp Messaging Bot Framework

Whatsapp BOT framework for Kulula airline: this was an R&D endeavour from Nacelle to build a white label framework for creating Whatsapp BOTs. The first prospect client being Kulula airline, allowing passengers to check-in to their flights by having a two-way conversation with the Whatsapp BOT with all their flight details i.e. starting the check-in process, sending their name and flight number, selecting a seat etc. Unfortunately the project was ended pre-maturely due to a lack of funding.



The framework had the following components: State Machine Generator & Runner: the framework was designed to be a white label product where we could design different process flows (dialog between BOT and user) without having to write code i.e. we would build up a decision tree/state machine by inserting data into a database e.g. creating the state machine's states, prompts, input takers, transitions etc. The framework would have a state machine runner that would execute the process flow setup in the database. I designed and completely developed this framework for creating and running a decision tree/state machine while my colleague implemented the CRUD data layer and logging of requests and responses. A future feature that I never got a chance to complete was creating Whatsapp BOT Studio application allowing us to design the work flows with a UI by dragging and dropping components of the decision tree/state machine. I only got the time to implement a POC of this using WPF for a desktop app. Database used in this framework was SQL Server with Entity Framework Core for interacting with the database. .NET Core Web API: we used Clickatell's ONE API for sending and receiving Whatsapp messages. Thus we needed a REST API to receive the callbacks from Clickatell i.e. when a user responded to a Whatsapp message and/or to be notified of a Whatsapp message having been delivered and read. The entire BOT was housed in a .NET Core Web API to receive the Clickatell call backs and make web requests back to the Clickatell ONE API. I developed the call backs REST actions and web requests i.e. integration with Clickatell

Bayhack Attorneys - DecAnalyzer



An application that gathers and saves records of deceased people in South Africa. The South African government does not have a website/system for searching for deceased people, instead it publishes a Government Gazette in PDF format of all deceased people and their estates. A lawyer who's business relies on handling the estates for deceased people and claims wanted a system that automates the reading of these PDF files and saves the records into a database for searching and receiving notifications when people have died and their estates become available for lodging claims. The business case being that the lawyer only has a short time frame to make a claim against an estate once a person has died. If he misses that deadline he's not able to submit the claim and therefore he and his client will not be able to recoup any money that the deceased owed to them.

Components of the software: Console App: that parses the PDF files and saves the records in a SQL Server database. It uses Adobe Services for splitting and extracting data out of the pDF files. WCF REST and & MVC API: for querying data out of the database. ASP.NET MVC Core Website: website for users to interact with the data, search for records, manage Debtors, Clients, Estates etc.

Psion Cabster
